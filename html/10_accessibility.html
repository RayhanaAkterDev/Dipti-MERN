HTML Accessibility

Web accessibility ensures that websites can be used by everyone, including people with disabilities. HTML provides several built-in features that help make web pages more accessible.

Semantic HTML

Semantic elements describe the purpose of their content, improving accessibility for assistive technologies such as screen readers.

Examples include <header>, <nav>, <main>, <section>, <article>, and <footer>.

Alternative Text

Images should include the alt attribute so screen readers can describe the image when it cannot be seen.

Labels for Form Controls

Every form control should have an associated <label> to improve usability and accessibility.

Using aria-label

The aria-label attribute provides an accessible name when visible text is unavailable or insufficient.

Writing semantic HTML and providing meaningful text alternatives are the foundation of creating accessible web pages. In many cases, semantic HTML alone provides excellent accessibility without requiring additional ARIA attributes.

HTML accessibility includes many more techniques than the ones demonstrated in this example.

Some commonly used accessibility features include role, tabindex, aria-labelledby, aria-describedby, aria-required, aria-live, aria-hidden, and proper keyboard navigation.
